# Chrome Web Store Review Notes

本扩展是一个单一用途工具：用户打开 Bilibili 视频页、列表页里的视频或直播间后，点击扩展图标，扩展会解析当前页面并自动复制对应的 VRChat/VRC 播放地址。弹幕实验模式需要用户自己提供后端地址，用于把普通 B 站视频生成带弹幕 HLS 地址。

测试步骤：

1. 安装扩展。
2. 首次安装后会自动打开扩展内置的本地使用指南页，用户可点击上一页、下一页或左侧步骤切换，也可以点击页面里模拟的 Chrome 右上角小尾巴图标查看复制成功提示。
3. 打开 Bilibili 普通视频页，例如 `https://www.bilibili.com/video/BV...`。
4. 普通模式下点击扩展图标。
5. 当前网页顶部中上方出现“已复制 91biliplayer 播放地址啦”提示，包装播放地址已写入剪贴板。
6. 在普通视频页快速双击扩展图标，可复制 B 站临时直链。
7. 在普通视频分P页、带 `bvid`/`aid`/`oid` 的列表页或直播间点击扩展图标，会自动复制直链。
8. 在扩展图标菜单中点击“切换小尾巴配置”，可调整普通 / 弹幕模式、直链清晰度、后端地址和弹幕参数。
9. 弹幕模式需要用户自己部署兼容 `/api/jobs` 的后端。填写后端地址后，在普通 BV/av 视频页点击扩展图标，扩展会把当前视频 URL 发给该后端，等待任务完成后复制 `hlsUrl`。

权限用途：

- `activeTab`：只在用户点击扩展图标时读取当前标签页 URL，并允许本次点击后在当前页显示短暂提示。
- `clipboardWrite`：用户点击扩展图标时写入剪贴板。
- `contextMenus`：在扩展图标菜单中显示“切换小尾巴配置”，用于打开配置页。
- `scripting`：只在用户点击扩展图标后向当前标签页注入一次页面上下文读取、复制 fallback、2 秒提示脚本；弹幕模式下也用来从当前页请求用户自填后端。
- `storage`：保存普通 / 弹幕模式、清晰度、后端地址和弹幕参数。
- `https://api.bilibili.com/*`：请求 Bilibili API 获取视频信息和播放信息。
- `https://api.live.bilibili.com/*`：请求 Bilibili 直播 API 获取直播间信息和直播播放信息。

隐私说明：

- 不保存用户账号数据。
- 不读取或保存 Cookie。
- 不保存生成的链接。
- 不包含远程代码。
- 不向作者服务器上传数据。
- 弹幕模式只请求用户自己填写的后端地址。
- 首次安装指南为扩展内置本地页面，只加载扩展包内样式、脚本和图标，不加载远程内容。
- 普通视频单击会生成 `biliplayer.91vrchat.com` 的第三方包装播放地址，该服务不是本扩展作者提供。

已知限制：

- 普通模式支持普通 BV/av 视频页、分P、带视频 id 的列表页和 B 站直播间；番剧/课程暂未实现直链。
- 弹幕模式目前只支持普通 BV/av 视频页，需要用户自己的后端和 ffmpeg。
- B 站临时直链可能过期。
- VRChat 播放是否成功取决于播放器、网络环境、第三方解析服务、自建后端和视频权限。
